\subsection{Detailed Description}
A driver for the I\-C\-O\-M 910 radio. 

This class provides a hardware driver for the I\-C\-O\-M 910 series of radios. It is primarily responsible for controlling the radio and automatically correcting its frequency for doppler shifts. In addition, it provides a command handler capable of controlling some features of the I\-C\-O\-M 910.

\begin{DoxyNote}{Note}
This driver requires that Hamlib2 be installed and configured, including the Python bindings. 

This driver is not currently capable of sending or receiving data to or from the radio directly. It is designed for pipelines that also contain a T\-N\-C, which will serve as the pipeline's input and output device. 

This driver is currently setup to operate in half-\/duplex mode. In half-\/duplex mode, only the M\-A\-I\-N V\-F\-O is set and the uplink frequency is specified via a split frequency on V\-F\-O B. 

The I\-C\-O\-M 910 driver will work with the T\-N\-C driver to make sure that it doesn't change its frequency when the T\-N\-C is sending data to the radio. This will prevent the I\-C\-O\-M from entering an undefined state. 
\end{DoxyNote}

Prepares the radio for use by a new session by loading the necessary services and setting up Hamlib. 

\begin{DoxyNote}{Note}
This method loads the active 'tracker' service from the session pipeline. The driver will use this service to load the doppler shift multiplier for the active target. If a 'tracker' service can not be loaded, no doppler correction will be applied. 

This method loads the active 'tnc\-\_\-state' service from the session pipeline. The driver will use this service to make sure that it does not allow its uplink frequency to be changed if data is being transmitted. If this service can't be located, no such protection will be provided (possibly putting the radio into an undefined state).
\end{DoxyNote}

Updates the radio's frequencies when new doppler shift information is available. 

\begin{DoxyVerb} This inline callback receives new target position information (including a doppler correction) from a 'tracker' 
 service. The driver will use this information to periodically (as defined by doppler_update_frequency) update 
 the uplink and downlink frequencies on the radio.
\end{DoxyVerb}


\begin{DoxyNote}{Note}
Because this method uses the set\-\_\-tx\-\_\-freq command, it will only update the uplink frequency if the radio has not recently transmitted. See the set\-\_\-tx\-\_\-freq command method for more information.
\end{DoxyNote}

Detailed Description

This test suite tests the functionality of the command parser (CommandParser), default Command class, and command Resource individually and holistically.

The functionality of individual commands (including system commands) is tested in the test suite for the containing command handler.

def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_metadata_errors (   self)

Test that the command metadata generation function correctly returns errors when appropriate.

If there are some subtle errors that don't get detected by the build_metadata_dict function, they will be handled when the command is processed.

def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_parser_failed_command (   self)

This test verifies that the command parser correctly returns an error response when a command fails (i.e.

raises an exception). In addition, it tests that the parser defaults to requiring an active user session if the command doesn't provide any meta-data.

Note
This test checks the "default to requiring an active session" condition successfully because the default mock session coordinator always returns 1 active mock session for any user ID.
def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_parser_invalid_schema (   self)

This test verifies that CommandParser correctly returns an error when an invalid command is submitted to it (i.e.

a command that doesn't conform to the schema).

def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_parser_successful_command_from_dict (   self)

This test verifies that the command parser allows a user to execute a command (that they have permission to) initially supplied as a dictionary.

This feature is used by pipeline setup commands.

def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_parser_successful_kernel_command (   self)

This test verifies that the command parser can correctly execute a command in kernal mode.

That is, a command that is not associated with any particular user and immune from session and permission requirements.

def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_parser_successful_system_command (   self)

This test verifies that the command parser allows a user to execute a command (that they have permission to) and returns the correct response.

In addition, it also tests that the user can execute commands that don't require an active session.

def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_parser_unrecognized_command (   self)

This test ensures that the command parser correctly rejects unrecognized commands.

In addition, it verifies the functionality of the optional CommandError exception which allows additional meta-data to be embedded with the error.

def hwm.command.tests.test_command_infrastructure.TestCommandInfrastructure.test_parser_unrecognized_kernel_command (   self)

Test that kernel mode commands correctly generate error responses for invalid commands.

In this case, an unrecognized command will be used to trigger the error.
